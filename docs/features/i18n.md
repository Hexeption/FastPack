# Multi-language support

The GUI displays all text in the user's chosen language. The setting persists across restarts.

## Usage

Open Preferences (Edit → Preferences…) and select a language from the General tab. The UI switches immediately — no restart needed.

Supported languages: English, Français, Español, Deutsch, Italiano, Português, 日本語, 中文（简体）, 한국어.

## Technical notes

Translations live in `crates/fastpack-gui/locales/` as JSON files, one per locale (`en.json`, `fr.json`, etc.). The `rust-i18n` crate loads them at compile time via `rust_i18n::i18n!("locales")` in `lib.rs`.

The active locale is set with `rust_i18n::set_locale(code)` and stored in `prefs.toml` as a `language` field.

All user-facing strings use the `t!("key")` macro. Named placeholders use the `%{name}` syntax from `rust-i18n`:

```rust
t!("log.pack_result",
    sprites = count,
    w = width,
    h = height,
    sheets = n,
    plural = if n == 1 { "" } else { "s" },
    aliases = alias_count,
    overflow = overflow_count,
)
```

Technical identifiers (PNG, JPEG, MaxRects, ShortSideFit, Scale2x, etc.) are not translated. They stay in English across all locales.

Collapsing section headers use a stable string key for their egui ID, separate from the translated label, so collapse state does not reset when the language changes.

## Adding a new language

1. Create `crates/fastpack-gui/locales/<code>.json` with all keys from `en.json` translated.
2. Add a new variant to the `Language` enum in `preferences.rs`.
3. Add `code()` and `display()` match arms, and append the variant to `Language::ALL`.
